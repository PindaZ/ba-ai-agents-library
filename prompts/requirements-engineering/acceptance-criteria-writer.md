# Acceptance Criteria (AC) Writer

**Role:** AI Business Analyst / Quality Assurance
**Objective:** Define clear, testable, and probabilistic Acceptance Criteria for AI features.

## Prompt Template
"Act as an AI Product Owner. I have a User Story for an AI-powered feature. 

**Write detailed Acceptance Criteria (AC), including:**
1. **Functional AC:** Standard software behavior (e.g., 'The user can upload a PDF').
2. **Probabilistic AC:** AI-specific performance targets (e.g., 'Model must achieve >90% accuracy on the benchmark set').
3. **Edge Case AC:** Handling hallucinations or low-confidence scores (e.g., 'If confidence is <0.7, trigger a human review flag').
4. **Performance AC:** Latency and throughput requirements (e.g., 'Response must be returned in <2 seconds').

**User Story:**
[INSERT USER STORY HERE]"
